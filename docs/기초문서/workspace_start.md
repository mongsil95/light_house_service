## 📌 핵심 원칙

### DB 작업 분리

이 프로젝트에서는 **프론트엔드 팀이 DB를 직접 다루지 않습니다**.

- ✅ **프론트엔드 팀**: UI/UX 구현, API 호출, 상태 관리
- ✅ **DB 팀**: 데이터베이스 설계, API 엔드포인트 구현, 서버 로직

### DB 관련 작업이 필요한 경우

모든 DB 관련 작업은 아래 문서에 **TODO 태그**를 사용하여 기록합니다:

📄 **[데이터베이스\_설계.md](./데이터베이스_설계.md)**

```markdown
- [ ] TODO: 사용자 테이블 생성 (users)
- [ ] TODO: 반려해변입양 API 엔드포인트 구현
```

## 🚀 작업 시작하기

### 1. 의존성 설치

```bash
npm install
```

### 2. 개발 서버 실행

```bash
npm run dev
```

### 3. 문서 확인

- **기술스택**: [기술스택.md](./기술스택.md)
- **DB 설계**: [데이터베이스\_설계.md](./데이터베이스_설계.md)
- **벤치마킹**: [배민CEO*사이트*분석.md](../벤치마킹/배민CEO_사이트_분석.md)

## ⚠️ 주의사항

1. **DB 직접 접근 금지**: 절대 DB에 직접 연결하거나 쿼리를 작성하지 마세요
2. **TODO 태그 사용**: 모든 DB 작업 요청은 데이터베이스\_설계.md에 기록
3. **API 명세 확인**: 데이터베이스\_설계.md에 정의된 API 엔드포인트만 사용
4. **팀 협업**: DB 관련 질문은 DB 팀에 문의

## 📝 작업 흐름

```
1. 기능 개발 필요
   ↓
```
