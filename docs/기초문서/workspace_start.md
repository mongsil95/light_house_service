## 📌 핵심 원칙

### 코드 스플릿 규칙

**파일당 최대 700줄 제한**을 준수하여 코드 가독성과 유지보수성을 향상시킵니다.

#### 분할 전략

1. **컴포넌트 분리**
   - 큰 컴포넌트는 작은 서브 컴포넌트로 분리
   - 재사용 가능한 UI 요소는 별도 파일로 추출
   - 예: `components/[feature]/[ComponentName].tsx`

2. **데이터 분리**
   - 상수, 타입, 인터페이스는 별도 파일로 관리
   - 예: `lib/[feature]Data.ts`, `types/[feature].ts`

3. **로직 분리**
   - 유틸리티 함수는 `lib/utils.ts` 또는 별도 파일로 분리
   - 커스텀 훅은 `hooks/use[HookName].ts`로 분리

4. **섹션 컴포넌트화**
   - 페이지 내 섹션이 많을 경우 각 섹션을 별도 컴포넌트로 분리
   - 예: `components/[page]/[SectionName]Section.tsx`

#### 700줄 초과 시 대응

```bash
# 파일 라인 수 확인
wc -l app/page.tsx

# 700줄 이상이면 즉시 분할
```

**분할 순서:**
1. 데이터 배열/객체 → `lib/[name]Data.ts`
2. 타입 정의 → `types/[name].ts`
3. 유틸리티 함수 → `lib/[name]Utils.ts`
4. UI 섹션 → `components/[page]/[Section].tsx`

### DB 작업 분리

이 프로젝트에서는 **프론트엔드 팀이 DB를 직접 다루지 않습니다**.

- ✅ **프론트엔드 팀**: UI/UX 구현, API 호출, 상태 관리
- ✅ **DB 팀**: 데이터베이스 설계, API 엔드포인트 구현, 서버 로직

### DB 관련 작업이 필요한 경우

모든 DB 관련 작업은 아래 문서에 **TODO 태그**를 사용하여 기록합니다:

📄 **[데이터베이스\_설계.md](./데이터베이스_설계.md)**

```markdown
- [ ] TODO: 사용자 테이블 생성 (users)
- [ ] TODO: 반려해변입양 API 엔드포인트 구현
```
